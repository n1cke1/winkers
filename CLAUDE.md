# Winkers

## Architecture

- `src/winkers/models.py` ‚Äî Pydantic models: Graph, FunctionNode, FileNode, CallEdge, ImportEdge
- `src/winkers/parser.py` ‚Äî TreeSitterParser (tree-sitter 0.25 API, QueryCursor)
- `src/winkers/graph.py` ‚Äî GraphBuilder: parse all files ‚Üí build Graph
- `src/winkers/resolver.py` ‚Äî CrossFileResolver: raw calls ‚Üí CallEdges with confidence
- `src/winkers/store.py` ‚Äî GraphStore: save/load .winkers/graph.json
- `src/winkers/semantic.py` ‚Äî SemanticLayer: Claude API enrichment ‚Üí .winkers/semantic.json
- `src/winkers/ui_map.py` ‚Äî UIMap: scan HTML/Jinja2 templates ‚Üí link Flask routes to UI elements (panels, tables, forms, buttons, inputs, tabs, indicators)
- `src/winkers/languages/` ‚Äî language profiles: Python, TypeScript, JavaScript, Java, Go, Rust, C#
- `src/winkers/mcp/` ‚Äî MCP server. `tools/` is a package: one module per tool (`orient.py`, `scope.py`, `convention_read.py`, `rule_read.py`, `before_create.py`, `browse.py`, `impact_check.py`, `session_done.py`) + `_common.py` (shared loaders + graph helpers). `find_work_area.py` lives here too but is a private helper invoked from `orient` ‚Äî not registered as a public MCP tool.
- `src/winkers/embeddings/` ‚Äî BGE-M3 via ONNX-INT8 (`Xenova/bge-m3`, ~1.1 GiB resident) + on-disk index (`.winkers/embeddings.npz`). Background preload on `serve` start, bounded `wait_for_preload(timeout)` rides out warmup. Set `WINKERS_USE_LEGACY_ST=1` to fall back to the float32 sentence-transformers stack.
- `src/winkers/target_resolution.py` ‚Äî intent categorization (create / change / unknown) + regex target extraction from graph's name dictionary
- `src/winkers/value_locked.py` ‚Äî AST detector for module-level literal collections (set/frozenset, ‚â§64 values), their consumer functions, and caller literal-arg usage. Powers `value_locked` warnings in scope/before_create/impact_check.
- `src/winkers/impact/` ‚Äî pre-computed impact analysis: models (`ImpactReport`, `CallerClassification`), `ImpactStore` (load/save/prune `.winkers/impact.json`), combined LLM prompt (intent + impact in one call), `ImpactGenerator` (FunctionContext, content_hash cache, batch concurrency, progress bar). Runs at `winkers init` when Claude API is configured.
- `src/winkers/hooks/` ‚Äî Claude Code hooks: prompt_enrich, pre_write, post_write, session_start, stop_audit (+ legacy session_audit module kept for migration sweeps).
- `src/winkers/intent/` ‚Äî LLM intent generation: provider.py (Ollama/API/None), eval_cli.py
- `src/winkers/detection/` ‚Äî duplicates.py (AST hash, name similarity), impact.py (signature diff)
- `src/winkers/session/` ‚Äî state.py (SessionState, WriteEvent, Warning, SessionStore)
- `src/winkers/search.py` ‚Äî function search by intent (tokenizer, stemmer, token cache); used by `before_create`, no longer surfaced as a CLI command
- `src/winkers/dashboard/` ‚Äî aiohttp + Cytoscape.js browser dashboard
- `src/winkers/cli/` ‚Äî CLI (click). `main.py` is a thin orchestrator (~110 LoC). Concrete commands live in `cli/commands/<name>.py`. Subgroups: `cli/hook_group.py` (Claude Code hooks), `cli/debug_group.py` (single-unit re-author + intent eval). Init's pipeline helpers live in `cli/init_pipeline/` (units, semantic, install, bootstrap, pipelines).

## Commands

```bash
.venv/bin/pytest tests/ -v            # run tests
.venv/bin/ruff check src/ tests/      # lint
winkers init                          # build graph + semantic + rules audit + hooks
winkers init --no-semantic            # graph only, no API call
winkers init --force/-f               # force re-enrichment even if graph unchanged
winkers init --yes/-y                 # auto-accept all rule proposals (no interactive review)
winkers init --ollama gemma3:4b       # use local Ollama for intent generation
winkers init --no-llm                 # skip intent generation
winkers init --with-units             # description-first units pipeline + BGE-M3 embeddings
winkers serve                         # MCP server (stdio)
winkers dashboard                     # browser graph
winkers doctor                        # health check (deps, graph, API key, MCP wiring)
winkers debug intent-eval --sample 20 --json   # evaluate intent quality
winkers debug describe-fn <fn_id>     # re-author one function unit (prompt iteration)
```

## Rules

- All models: Pydantic BaseModel with type hints
- Graph helpers: use `graph.is_locked(fn_id)`, `graph.callers(fn_id)`, `graph.callees(fn_id)` ‚Äî methods on Graph
- tree-sitter: use QueryCursor for captures/matches (tree-sitter 0.25 API). Do NOT use tree-sitter-languages
- Language packages: individual (tree-sitter-python, tree-sitter-java, etc.)
- Functions < 40 lines, one class per concern
- No dynamic imports

## Three-file model

- **graph.json** ‚Äî structure: files, functions (with ast_hash, intent, secondary_intents), call edges, import edges, complexity
- **semantic.json** ‚Äî intent: zone intents, data_flow, domain_context, checklist (generated by Claude API, one call)
- **rules.json** ‚Äî conventions/rules (ConventionRule with stats; generated by LLM audit + detectors, accepted by user). Coherence rules: sync_with, fix_approach (sync/derived/refactor)
- **impact.json** ‚Äî per-function LLM-assessed risk: risk_level, caller_classifications, safe/dangerous ops, action_plan. Combined with intent in one LLM call at init time; cached by content_hash.
- **units.json + embeddings.npz** ‚Äî per-unit LLM descriptions (function/class/attribute/value/template/data) + BGE-M3 vectors. Built by `init --with-units`. Read by `orient(task=‚Ä¶)` for `semantic_matches`.
- **expressions.json** ‚Äî AST expression-uses index for `value_locked` impact (Path 2 of literal-blind fix). Built by `init`.
- **session.json** ‚Äî ephemeral: writes, warnings, session_done_calls (lives during MCP session, gitignored)
- **config.toml** ‚Äî intent provider settings: provider, model, prompt_template

Zero duplication. Graph = facts. Semantic = meaning. Rules = standards. Units + embeddings = retrieval surface.

## Semantic schema

- `ZoneIntent` ‚Äî why zone exists + wrong_approach
- `MonsterFile` ‚Äî sections (prefix, purpose, count) + where_to_add
- `SemanticLayer` ‚Äî data_flow, domain_context, zone_intents, monster_files, new_feature_checklist
- `proposed_rules` ‚Äî returned by SemanticEnricher but NOT saved to semantic.json ‚Üí goes to rules.json after user confirmation

`conventions[]` and `constraints[]` are removed ‚Äî replaced by rules.json.

## MCP tools (8 total)

- `orient(task, include, zone?, min_callers?, k?)` ‚Äî single entry point. `task` is mandatory; orient runs the per-unit semantic search internally and returns `semantic_matches` alongside the requested sections (`map`, `conventions`, `rules_list`, `functions_graph`, `hotspots`, `routes`, `ui_map`). Shows session status if active.
- `browse(zone?, file?, min_callers?, limit?, offset?)` ‚Äî mid-level function inventory between orient and scope. Returns `"file::fn (callers) ‚Äî intent"` strings, paginated. Intent omitted when null. With `file=`, caller call-sites are inlined under each fn as `"  ‚Üê caller_file:line  expression"` ‚Äî browse becomes a one-shot "who calls what I'm about to edit" view. Use after orient to scan what functions exist and what they do before picking a target.
- `scope(function?, file?)` ‚Äî deep context: callers, callees, related_rules, recent changes. Function-level response also returns pre-computed `impact` (risk_level/safe+dangerous ops/classified callers/action_plan) and `similar_logic` (functions sharing `secondary_intents`) when impact.json exists.
- `convention_read(target)` ‚Äî zone name as in conventions (e.g. "app.py") / "data_flow" / "domain_context" / "checklist"
- `rule_read(category)` ‚Äî all rules for a category + wrong_approach; use orient rules_list for available categories
- `before_create(intent, zone?)` ‚Äî classifies intent (create / change / unknown), resolves targets from graph. **Prefer explicit markers in intent**: `fn_name()`, `Class.method()`, paths (`app/repos/invoice.py`), or `file.py::fn` ‚Äî parser resolves those first, fuzzy matching is fallback. For `change` returns adaptive payload (`files` block with coupling / `importing_files` / `migration_cost` / `direct_caller_files` (fn-level) / `locked_fns`, `functions` block with affected_fns + caller expressions + `risk_level` + `dangerous_operations`, `similar_logic` warning when target shares `secondary_intents`). Test-path files filtered unless intent contains `tests/`/`test_`/`conftest`.
- `impact_check(file_path)` ‚Äî incremental graph update, impact analysis, coherence check, session state (renamed from `after_create` in 0.8.1)
- `session_done()` ‚Äî optional final session audit across all writes; anti-loop (FAIL max once)

`find_work_area` is no longer registered as a public tool ‚Äî its functionality is bundled into `orient(task=‚Ä¶)`. The implementation stays as a private helper used by orient.

## Architectural context (Winkers)

[Winkers](https://github.com/n1cke1/winkers) MCP: function-level dependency graph, zones, rules. Use before non-trivial edits.

### Workflow

1. `orient` with `task: "<what you were asked to do>"`, `include: ["map", "conventions", "rules_list"]` ‚Äî zones, hotspots, data flow, zone intents, coding rules, **plus** `semantic_matches` (top-K relevant units ranked by embedding similarity against your task). `task` is **mandatory**. **First call.** Bundles the per-unit semantic search that was previously a separate `find_work_area` step.
2. `browse` with `zone` or `file` ‚Äî mid-level inventory: function list with LLM intents (`"file::fn (callers) ‚Äî intent"`). With `file=`, caller call-sites are inlined under each fn (`"  ‚Üê caller_file:line  expression"`) so you see who invokes what before editing. Use to pick a target before deep-dive.
3. `before_create` with `intent: "<what you're about to change>"` ‚Äî matches, affected callers (expressions + risk), `similar_logic`. **Prefer explicit targets** (`fn_name()` / `Class.method()` / `Class.attribute` / path / `file.py::fn`). **One call per concrete change** ‚Äî batched intents dilute signal. **Before writing any code.**
4. Write / edit code.
5. `impact_check` with `file_path: "<path>"` ‚Äî graph update + duplicate detection + broken import check. Auto via hook in Claude Code; call explicitly in other agents.

### Task / intent formation rules

`task` (orient) and `intent` (before_create) are **two distinct inputs**:
- **`task`** ‚Äî task-level, broad: what was assigned to you. Set once per session.
- **`intent`** ‚Äî change-level, narrow: what you're about to modify right now. Set per concrete change.

Both follow the same structural rules. A useful task/intent is:

| Component | Required? | Example |
|-----------|-----------|---------|
| Verb-first | required | `create` / `change` / `fix` / `add` / `refactor` / `extract` / `remove` / `rename` / `audit` / `simplify` |
| Target if applicable | for `intent`: strongly preferred; for `task`: optional | `Class.method()`, `Class.attribute`, `file.py::fn`, path |
| Goal in one phrase | required | what should become, not how to get there |
| One concern | required | no `and` / `&` / multi-task lists |

**‚úÖ Good:**
- `simplify invoice statuses from 6 to 3`
- `fix Client.invoices relationship cascade`
- `add soft-delete to all financial repos`
- `extract date utilities from app/services/billing.py`
- `audit soft-delete consistency across repos`

**‚ùå Bad:**
- `improve invoice handling` ‚Äî no concrete verb, no scope
- `invoices` / `statuses` ‚Äî bare noun
- `fix bug X and add feature Y` ‚Äî multi-task, dilutes intent fulfillment audit
- `rewrite using Pydantic v2` ‚Äî implementation-first, goal lost
- `make it better` / `refactor everything` ‚Äî no target, no verb specificity

`orient` returns `task_warnings` (non-blocking) when the task is structurally weak: < 3 words, multi-task markers, or no semantic_matches scoring above 0.5. Treat warnings as guidance, not gates.

### On demand

| Tool | When |
|------|------|
| `browse` with `zone` / `file` / `min_callers` / `limit` / `offset` | list functions + intents, paginated |
| `scope` with `file` or `function` | coupling, caller expressions, `impact` (risk, safe+dangerous ops), `similar_logic` |
| `rule_read` with `category` | full rule text when the one-liner isn't enough |
| `orient` with `functions_graph` / `routes` / `hotspots` | deeper call-graph / endpoints / risk-ranked fns |
| `convention_read` with `target` | zone intent / data_flow / checklist |
| `session_done` | optional cross-file audit |

### Key concepts

- **locked** ‚Äî has callers; don't change signature without updating them.
- **free** ‚Äî no callers; modify freely.
- **value_locked** ‚Äî module-level literal set; removing a value breaks callers passing it as a literal.
- **risk_level** ‚Äî `low`/`medium`/`high`/`critical` per function from `scope.impact` / `hotspots`; heed `dangerous_operations` before editing.
- **secondary_intents** ‚Äî inline sub-task tags; `similar_logic` flags duplicated logic ‚Äî extract rather than duplicate.
- **direct_caller_files** vs **migration_cost** (`before_create.files`) ‚Äî `direct_caller_files` = files that actually *call* the fn being changed (tight surface). `migration_cost` = raw import-edge count (loose upper bound). Prefer `direct_caller_files` on fn-level intents.
- **route / http_method** ‚Äî HTTP-handler marker (Flask / FastAPI / Django / aiohttp). Inlined in `scope`, `browse` (`[METHOD /path]`), `hotspots`, `before_create.affected_fns`.
