# Winkers

## Architecture

- `src/winkers/models.py` — Pydantic models: Graph, FunctionNode, FileNode, CallEdge, ImportEdge
- `src/winkers/parser.py` — TreeSitterParser (tree-sitter 0.25 API, QueryCursor)
- `src/winkers/graph.py` — GraphBuilder: parse all files → build Graph
- `src/winkers/resolver.py` — CrossFileResolver: raw calls → CallEdges with confidence
- `src/winkers/store.py` — GraphStore: save/load .winkers/graph.json
- `src/winkers/semantic.py` — SemanticLayer: Claude API enrichment → .winkers/semantic.json
- `src/winkers/languages/` — language profiles: Python, TypeScript, JavaScript, Java, Go, Rust, C#
- `src/winkers/mcp/` — MCP server (tools: map, functions_graph, hotspots, scope, routes) + auto-rebuild on file changes
- `src/winkers/dashboard/` — aiohttp + Cytoscape.js browser dashboard
- `src/winkers/cli/` — CLI (click): init, serve, dashboard

## Commands

```bash
.venv/Scripts/pytest tests/ -v       # run tests
.venv/Scripts/ruff check src/ tests/ # lint
winkers init                          # build graph + semantic (one Claude API call)
winkers init --no-semantic            # graph only, no API call
winkers serve                         # MCP server (stdio)
winkers dashboard                     # browser graph
```

## Rules

- All models: Pydantic BaseModel with type hints
- Graph helpers: use `graph.is_locked(fn_id)`, `graph.callers(fn_id)`, `graph.callees(fn_id)` — methods on Graph
- tree-sitter: use QueryCursor for captures/matches (tree-sitter 0.25 API). Do NOT use tree-sitter-languages
- Language packages: individual (tree-sitter-python, tree-sitter-java, etc.)
- Functions < 40 lines, one class per concern
- No dynamic imports

## Two-file model

- **graph.json** — structure: files, functions, call edges, import edges, complexity
- **semantic.json** — intent: zone intents, constraints, conventions, checklist (generated by Claude API, one call)

Zero duplication. Graph = facts. Semantic = meaning.

## Semantic schema

- `ZoneIntent` — why zone exists + wrong_approach
- `MonsterFile` — sections (prefix, purpose, count) + where_to_add
- `SemanticLayer` — data_flow, domain_context, zone_intents, monster_files, new_feature_checklist
- `proposed_rules` — returned by SemanticEnricher but NOT saved to semantic.json → goes to rules.json after user confirmation

`conventions[]` and `constraints[]` are removed — replaced by rules.json.

## MCP tools (4 total)

- `orient(include, zone?, min_callers?)` — single entry point; include: "map", "conventions", "rules_list", "functions_graph", "hotspots", "routes"
- `scope(function?, file?)` — deep context: callers, callees, related_rules, recent changes
- `convention_read(target)` — zone name as in conventions (e.g. "app.py") / "data_flow" / "domain_context" / "checklist"
- `rule_read(category)` — all rules for a category + wrong_approach; use orient rules_list for available categories
